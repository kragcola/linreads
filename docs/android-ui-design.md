# LinReads Android UI 设计文档 v3.5

_更新：2026-06-19 · 设计语言：纸张质感优先 · 旧书的温度 · 去"刻板典雅"_

> **范围**：全界面视觉 + 设计系统。预览见 [ui-preview/index.html](ui-preview/index.html)。
> **落点**：token/组件归属 `:core:ui`（Layer 5）。规范文档,不含实现代码,实现需另行放行。

## 为什么推翻 v1/v2（根因诊断）

两版都失败,且都失败在同一处,所以这次换的是**做法**,不是配色:

- v1 暖白 + 暖木棕 = frontend-design 点名的 AI 默认之一（cream + terracotta）。
- v2 藏青墨 + 黄铜金 = AI 默认的另一个（near-black + 单一亮强调）。我躲开第一个却跳进第二个。
- **真正的病根**:两版都"先挑一套典雅配色 + 衬线 + 强调色"。这套公式本身就是模板。v2 的铜丝带、台灯 radial glow 是往公式上贴"典雅",所以更刻板、更冷,纸感反而没了——因为纸从来没真出现过,只是平涂色块在假装。

**v3 换的做法**:以**纸张材质**为主体,不是以配色为主体。
1. 纸是真的有纹理的——细微纤维颗粒、不均匀的柔光投影,而非一块死色。
2. 调色近乎单色:**纸 + 墨**,加一点点旧布封面的脏色,不要金、不要亮强调。
3. 温度和记忆点来自**手作的不完美**:夹在书里、稍微歪一点的纸书签;磨损的布面封面;不是来自一个"优雅"的强调色。
4. 气质是**翻旧书**,不是"高级藏书楼"。随意、有人味、被读过,而非博物馆陈列。

## 一、设计系统

### 1.1 材质优先（这是 v3 的核心，不是配色优先）

界面的主体是**纸**,不是色块。落地手段:

- **纸纹理**:背景不是纯色填充,叠一层极淡的纤维噪点 / 纸纹（实现用 `0.5%~1.5%` alpha 的 noise overlay 或 tiling 纹理图）。让大面积"纸"区域有微小的明暗起伏,不是死平。
- **柔光投影**:封面、卡片的投影是**柔的、暖的、略微不均匀**（多层 blur,低 alpha,偏暖灰),像实物在漫射光下,而非 Material 的均匀 elevation。
- **手作不完美**:书签纸条略歪 1-2°;封面布面有磨损暗角;分组标题下的"线"是手绘感的不规则细线,不是 1px 数学直线。
- **不要**:金属高光、radial 聚光灯、亮强调色、玻璃拟态 blur 玻璃板。这些都是"数码典雅",和纸感对立。

### 1.2 调色：纸 + 墨 + 一点脏（近单色）

不设"强调色"。颜色总数压到最少,靠**明度和材质**拉层级,不靠彩色点缀。

| Token | 值 | 说明 |
|-------|-----|------|
| `paper` | `#EDE6D6` | 主纸色,米白偏暖,带一点旧 |
| `paper.deep` | `#E2D9C4` | 凹陷区/次级纸(书架隔层) |
| `paper.shadow` | `#C9BFA6` | 纸与纸之间的暖灰阴影 |
| `ink` | `#2A2620` | 墨字,近黑带暖(非纯黑) |
| `ink.soft` | `#6E665A` | 次要字(作者/元信息),淡墨 |
| `cloth.red` | `#8A4A3C` | 旧布封面脏红(克制使用,做封面/书签布,不做 UI 强调) |
| `cloth.blue` | `#3E4A52` | 旧布封面灰蓝 |
| `cloth.green`| `#4A5240` | 旧布封面橄榄 |

> 关键:`cloth.*` 是**封面材质色**,不是 UI 主题色。UI 本身只有纸和墨。这就避免了"挑一个强调色往界面上点"的模板做法。
> 对比度:`ink #2A2620` on `paper #EDE6D6` ≈ 11.5:1,远超 AA。

#### 夜间
不做"深色典雅壳"。夜间是**关了大灯、留一盏暖台灯读旧书**:纸压暗成 `#2A2620` 的暖褐(不是冷黑/藏青),墨翻成 `#D8CFBC` 暖白纸色字。仍是"纸+墨",只是天黑了。

### 1.3 排印

- 正文:宋体（思源宋体 Regular）,18sp,行高 1.75,这部分 v2 没错,保留。
- 标题/书名:同一套宋体,**不**靠超大字号或特殊字重充"设计感"——旧书的书名页就是朴素的。22sp Medium。
- UI 文字(按钮/标签/导航):思源黑体 14sp,安静。
- 不追求戏剧性的 type scale。旧书的气质是克制,不是排版炫技。

### 1.4 间距与触感

- 间距 4/8/12/16/24dp,屏幕边距 18dp。
- 封面圆角 2dp（书是近乎方角的实物,大圆角立刻塑料感）。
- 卡片/纸面圆角 8dp。
- 触摸目标 ≥ 48dp。

## 二、界面设计

### 2.1 书架 Library（初始屏 · 克制隔板 + 整齐封面网格）

书架是**整齐对齐的封面网格,全部正面封面朝外**;每行书"搁在"一道**克制的窄隔板沿**上。隔板不是视觉主角——它只是一道细窄暖色板沿 + 书底投下的柔影,暗示"书搁在面上"。封面才是 hero,隔板退到背景。参考静读天下保留"书架"实体暗示,但**去掉厚重木纹大板**(那会喧宾夺主,把封面压下去)。进度用**封面底部细进度条**。

> **设计修正(v3.4,调用 design-audit + frontend-design)**:v3.3 的厚木板(13px 高、暖木渐变、亮前缘高光)抢了封面的视觉重心——板比书还显眼,违反"封面是书的脸""boldness 集中一处(歪书签)"。v3.4 把隔板压成 **4px 窄板沿 + 书底柔影**:质感靠"书在面上投影"传达,不靠木纹本身。木头退场,书归位。

```text
┌──────────────────────────────────┐
│ 〔纸纹背景,微噪点〕                 │
│  我的书           ⌕   ⚙           │  朴素宋体,不喊"藏书阁"
│                                  │
│  在读 ·······················     │  手绘感细线(略不规则)
│   ┌──┐                           │
│   │封│╮ 一张略歪的纸书签夹在里      │  ← 纸书签(唯一记忆点),
│   │面│┊ 《活着》余华  读到 67%      │     夹在在读书里
│   └──┘╯                          │
│                                  │
│   ┌──┐  ┌──┐  ┌──┐  ┌▦▦┐         │  ← 封面整齐排列,全朝外
│   │封│  │封│  │封│  │叠 │          │  ┌▦▦┐=合订(§2.1.2)
│   │▰▰│  │▰ │  │  │  │  │          │  封面底部细进度条
│  ‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗      │  ← 窄隔板沿(细+书底柔影,不喧宾)
│   ┌──┐  ┌──┐  ┌──┐  ┌──┐         │
│   │封│  │封│  │封│  │封│          │  行列对齐,等间距
│  ‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗‗      │  ← 下一道窄板沿
├──────────────────────────────────┤
│   书架       阅读        设置      │  文字导航,选中加墨点·
└──────────────────────────────────┘
```

温度与记忆点(单点 boldness,对齐 frontend-design):
- **唯一记忆点 = 夹在在读书里、稍歪的纸书签**。这是全屏唯一张力,周围(封面、隔板)全部克制,书签才跳出来。
- 背景纤维噪点纸 + **窄隔板沿**(4px 暖色细板 + 书底柔影,只作"搁置面"暗示,不抢戏)。
- 封面旧布/纸质感(`cloth.*` 脏色 + 磨损暗角 + 布纹),整齐对齐不抖动——封面是这屏的视觉主体。
- 进度:**封面底部细进度条**(墨色填充),不占额外行,不用圆角标。
- 文案朴素:"我的书""在读""书架",不端着。

### 2.1.1 无封面 fallback（全封面前提下）

不退回书脊。无封面的书显示一张**素封面**:旧纸/布底色(`paper.deep` 或一个稳定 `cloth.*`)+ 居中烫印书名 + 作者,边缘磨损暗角。与有封面的书同尺寸整齐并排,看上去是一本朴素精装本。

### 2.1.2 合订文件夹（叠放，封面数随书数）

把几本书归到一起 = **一叠摞着的书**(不是 OS 方形文件夹图标)。叠放层数**按组内书数动态显示,最多露出前 4 本**封面的边:

| 组内书数 | 显示 |
|---------|------|
| 2 本 | 顶 1 本完整封面 + 底 1 本露边(共 2 层) |
| 3 本 | 顶 + 2 层露边(3 层) |
| ≥ 4 本 | 顶 + 3 层露边(共 4 层,封顶,再多不加层) |

```text
  2 本        3 本        4 本及以上
  ┌───┐      ┌───┐       ┌───┐
 ┌┴──┐│     ┌┴──┐│      ┌┴──┐│
 │顶 │┘    ┌┴──┐├┘     ┌┴──┐├┘
 └───┘     │顶 │┘     ┌┴──┐├┘
           └───┘      │顶 │┘
                      └───┘
```

- 顶层 = 该组**最上面一本(或最近读)**的封面正面朝外;底下各层取后续书封面的**边**,向右下错开 + 渐深阴影,模拟摞着的书。
- 露出的层数 = `min(书数, 4)`,书越多叠越厚(到 4 封顶),厚度即数量的直觉表达。
- **不显示"· N 本"那行字**——叠的厚度已经表达数量,文字多余。组名(宋体)正常显示在封面下方,和单本书的书名位置一致。
- 与单本书同尺寸整齐并排,靠"层叠的边"区分单本 vs 一叠。
- 交互:点击展开成该组封面墙(同书架样式子页,非弹窗);长按重命名/拆分。
- 错开方向统一**右下**(顺左上翻页直觉),阴影暗示厚度。

## 三、多分辨率 / 多设备适配

Android 设备从 4" 手机到 13" 平板、折叠屏都要覆盖。原则:**书架是"自适应的封面网格",列数随可用宽度变,封面物理观感(单本封面的视觉尺寸)保持稳定**——不是把封面拉大填满平板,而是在更宽的屏上摆更多本,像更大的书架。

### 3.1 断点（按 Material3 Window Size Class）

| Window Class | 宽度 dp | 设备 | 书架列数 | 布局策略 |
|--------------|---------|------|---------|---------|
| Compact | < 600 | 手机竖屏 / 小折叠外屏 | 3 列 | 单栏,底部导航 |
| Medium | 600–839 | 手机横屏 / 7-8" 平板 / 折叠内屏 | 4 列 | 单栏,导航转**侧轨 NavigationRail** |
| Expanded | ≥ 840 | 10"+ 平板 / 折叠大屏 / 桌面窗 | 5+ 列 | 侧轨 + 阅读器可**双栏**(目录/正文) |

> 列数是下限示意;实际用**自适应网格**(下条)按"目标封面宽度"算,断点只决定导航形态与是否双栏。列数刻意保守(宁少勿挤),让封面更大、间距更松,书架透气而非塞满。

### 3.2 自适应封面网格（核心机制）

不写死列数,定义**单本封面的目标宽度**,让列数 = `floor(可用宽 / 目标宽)`,余量均分到间距。Compose 用 `LazyVerticalGrid(GridCells.Adaptive(minSize = coverTargetWidth))`。

| 维度 | 取值 | 理由 |
|------|------|------|
| 封面目标宽 `coverW` | 手机 116dp · 平板 132dp(随 density 不变 dp) | 封面更大更舒展,宽屏摆更多本但不挤 |
| 封面宽高比 | 固定 2:3 | 书的比例恒定,不随屏变形 |
| 网格间距 | 18dp(Compact)→20dp(Medium)→24dp(Expanded) | 透气优先,书与书之间留足呼吸 |
| 最大内容宽 | Expanded 下书架内容区限 1200dp 居中 | 超宽屏不让网格拉到天涯,两侧留纸边 |

阅读器正文同理:`maxLineWidth = 680dp` 始终生效(§1.3),平板/横屏下正文居中,两侧是纸的留白,不铺满——宽屏读书的行宽控制比手机更关键。

### 3.3 折叠屏 / 横竖切换

- **配置变更不丢状态**:列数/布局变化走 Compose 重组 + `rememberSaveable` 保留滚动位置与在读态(对齐 v4 §7.2 进程死亡恢复链,横竖屏只是轻量 config change)。
- **折叠铰链**:Expanded 双栏时避开 hinge(用 `WindowLayoutInfo` 的 fold bounds 作分栏位)。Phase 1 可不处理,留接口。
- **在读 hero**:Compact 横置(封面左、信息右);Expanded 下可与书架并列或置顶通栏。

### 3.4 密度 / 字体缩放

- 所有尺寸用 **dp/sp**,不用 px;封面、间距随 density 自动缩放,物理观感一致。
- 尊重系统字体缩放:正文 sp 随系统放大,但**最大行宽 dp 不变** → 字大了自动减少每行字数,不破版。
- 超大字体无障碍模式:书架书名最多 2 行截断 + 省略,不挤爆网格。

### 3.5 验证矩阵（实现时）

| 设备类 | 验证 |
|--------|------|
| Compact 手机 | 3 列网格,底部导航,封面 116dp |
| Medium 平板/横屏 | 4 列,NavigationRail,封面 132dp |
| Expanded 大平板 | 5+ 列,内容限宽居中,阅读器双栏 |
| 折叠屏展开/折叠 | 状态不丢,布局平滑切换 |
| 字体 200% | 正文不破版,书名截断 |

### 2.3 阅读器 Reader

阅读区本就该是纸。三主题都是"不同光线下的同一张纸",不是换皮肤。

```text
chrome 收起(默认):            chrome 呼出:
┌──────────────────┐         ┌──────────────────┐
│〔纸纹·微噪点〕      │         │ ←  活着 第三章    │ 顶栏=纸,压暗一线
│                  │         │                  │
│ 第三章            │         │ (正文,纸压暗一档)  │
│                  │         │                  │
│ 正文宋体,18sp     │         │  Aa   光   目录   │ 字号/亮度/目录
│ 行高1.75,墨字     │         │  ━━━━━╸──── 47%   │ 进度=墨线,把手墨点
│ 纸纹背景,无遮挡    │         │                  │
└──────────────────┘         └──────────────────┘
```

- "主题"按钮叫"光"——日间 / 夜灯 / 暖黄,语义是调光线,不是选色板。
- 进度条是一道**墨线**,把手是墨点,不是金属/亮色滑块。
- chrome 是同一张纸压暗,不是浮起的玻璃板。

## 四、无障碍

- 纸纹噪点 alpha 极低,不影响 `ink/paper` ≈ 11.5:1 对比度(过 AA)。
- 全交互元素 `contentDescription`;纸书签朗读"在读 活着,余华,67%"。
- 触摸目标 ≥ 48dp;字号系统缩放 + app 内调,正文上限 24sp。
- 进度墨线提供无障碍滑动语义。
- reduce-motion:关掉书签轻微飘动等微动效,纹理静态保留。
